# ТЗ: «Кико» (キコ) — личный трекер вопросов первой линии

> PWA-приложение для одного пользователя: мгновенный захват вопросов от пользователей, статусы как в тикетнице, настойчивые push-напоминания про открытые вопросы, автоматический PDF-отчёт «что сделано» для удалёнки.

---

## 1. Контекст и цель

Пользователь работает бизнес-аналитиком и сидит на первой линии поддержки. Вопросы приходят по телефону/лично, часть решается сразу, часть откладывается «на потом» — и теряется. Нужен инструмент, который:

1. Позволяет **зафиксировать вопрос за 5 секунд**, не отрываясь от разговора.
2. **Не даёт забыть** про открытые вопросы — пингует пушами, пока вопрос не закрыт.
3. Ведёт **лог всей работы** и генерирует **красивый PDF-отчёт** за период (день/неделя) для руководителя.

Продукт строго персональный: один пользователь, без регистрации/логина в MVP (защита — простой PIN или magic link Supabase, см. раздел 9).

---

## 2. Платформа и ключевые ограничения

- **Формат:** PWA (Progressive Web App), устанавливается на iPhone через Safari → «Поделиться» → «На экран "Домой"».
- **Критично:** Web Push на iOS работает **только** из установленной PWA (iOS ≥ 16.4). Из вкладки Safari пуши не работают. В UI обязателен онбординг-экран, который определяет, запущено ли приложение в standalone-режиме (`window.matchMedia('(display-mode: standalone)')`), и если нет — показывает инструкцию по установке с картинками.
- После установки и первого запуска — экран запроса разрешения на уведомления (`Notification.requestPermission()` должен вызываться **только по явному тапу пользователя на кнопку**, иначе iOS молча откажет).
- HTTPS обязателен — деплой на Vercel закрывает это из коробки.

## 3. Стек

| Слой | Технология |
|---|---|
| Frontend | React 18 + Vite + TypeScript + Tailwind CSS |
| PWA | `vite-plugin-pwa` (manifest + service worker, стратегия `injectManifest` — нужен кастомный SW для обработки push) |
| Backend / БД | Supabase (Postgres, Edge Functions, `pg_cron`) |
| Push | Web Push API (VAPID-ключи), отправка из Supabase Edge Function через библиотеку `web-push` (Deno-совместимый аналог: `jsr:@negrel/webpush` или ручная реализация через `crypto.subtle`) |
| PDF | `@react-pdf/renderer` — генерация на клиенте, скачивание/шаринг через `navigator.share()` |
| Хостинг | Vercel |
| Опционально (v2) | Anthropic API для AI-парсинга текста вопроса |

## 4. Схема БД (Supabase)

```sql
-- Вопросы (тикеты)
create table questions (
  id uuid primary key default gen_random_uuid(),
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),
  closed_at timestamptz,

  -- Контент
  raw_text text not null,              -- что ввёл пользователь как есть
  author text,                          -- кто спросил («Иванова, бухгалтерия»)
  summary text,                         -- краткая суть (в MVP = raw_text, в v2 заполняет AI)

  -- Классификация
  status text not null default 'new'
    check (status in ('new', 'in_progress', 'waiting', 'closed')),
  priority text not null default 'normal'
    check (priority in ('urgent', 'normal', 'low')),
  waiting_for text,                     -- от кого ждём ответа (для статуса waiting)

  -- Напоминания
  remind_at timestamptz,                -- следующее напоминание
  remind_interval_minutes int not null default 240, -- шаг эскалации
  snooze_count int not null default 0,

  -- Итог (для отчёта)
  resolution text                       -- как решили / что ответили
);

create index idx_questions_open on questions (status, remind_at)
  where status != 'closed';

-- Push-подписки (одна на устройство, но храним массивом на случай iPad и т.п.)
create table push_subscriptions (
  id uuid primary key default gen_random_uuid(),
  created_at timestamptz not null default now(),
  endpoint text not null unique,
  p256dh text not null,
  auth text not null,
  user_agent text
);

-- Лог активности (основа отчёта)
create table activity_log (
  id uuid primary key default gen_random_uuid(),
  created_at timestamptz not null default now(),
  question_id uuid references questions(id) on delete cascade,
  event text not null
    check (event in ('created', 'status_changed', 'snoozed', 'closed', 'note_added')),
  detail jsonb                          -- {"from": "new", "to": "in_progress"} и т.п.
);

-- Триггер updated_at
create or replace function touch_updated_at() returns trigger as $$
begin new.updated_at = now(); return new; end;
$$ language plpgsql;

create trigger questions_touch before update on questions
  for each row execute function touch_updated_at();
```

RLS: включить на всех таблицах. В MVP — политика на роль `anon` с проверкой кастомного заголовка нельзя, поэтому: либо всё через Edge Functions с service key (клиент → Edge Function → БД), либо Supabase Auth с одним пользователем (magic link) и политики `auth.uid() = <мой uid>`. **Рекомендуемый вариант — Auth с magic link**, это 15 минут настройки и честная защита данных.

## 5. Логика статусов и напоминаний

### Статусы

```
new ──────────► in_progress ──────────► closed
 │                   │  ▲
 │                   ▼  │
 └────────────► waiting ┘   (жду ответа от кого-то)
```

- **new** — только зафиксирован. Дефолтное `remind_at`: `urgent` → +2 часа, `normal` → следующее утро 09:30, `low` → +2 дня 09:30.
- **in_progress** — взят в работу. Напоминание каждые 4 часа в рабочее время.
- **waiting** — ждём третью сторону. Напоминание раз в день в 10:00: «Пни {waiting_for} по вопросу "{summary}"».
- **closed** — обязательное поле `resolution` (одной строкой, «что ответил/сделал»). Попадает в отчёт.

### Правила напоминаний

- **Рабочее окно:** пуши только пн–пт 09:00–19:00 (настройка в приложении, хранится в `localStorage` + дублируется в таблице настроек или в Edge Function константой для MVP). Если `remind_at` выпадает на ночь/выходные — переносим на ближайшее рабочее утро.
- **Эскалация:** каждый snooze увеличивает `snooze_count`; после 3 снузов интервал сокращается вдвое (напоминание становится настойчивее, а не реже) и в тексте пуша появляется «⚠️ Уже {n} раз откладывал».
- **Утренний дайджест:** один пуш в 09:15 каждый рабочий день: «Открыто: 4 · Срочных: 1 · Ждёшь ответа: 2». Тап открывает главный экран.
- **Вечерний чек (опция, выкл. по умолчанию):** в 18:30 «Закрой хвосты: 2 вопроса без движения сегодня».

### Механика (Supabase)

- `pg_cron` job каждые 15 минут вызывает Edge Function `send-reminders`.
- Функция: выбирает `questions where status != 'closed' and remind_at <= now()`, группирует, шлёт Web Push по всем подпискам, сдвигает `remind_at` на следующий интервал, пишет событие в `activity_log`.
- Отдельный cron на 09:15 будних дней → Edge Function `morning-digest`.
- VAPID-ключи — в секретах Supabase (`supabase secrets set`). Приватный ключ никогда не попадает на клиент. Публичный — в env фронта (`VITE_VAPID_PUBLIC_KEY`).
- Мёртвые подписки (HTTP 404/410 от push-сервиса Apple) — удалять из `push_subscriptions`.

### Service Worker (клиент)

```js
self.addEventListener('push', (e) => {
  const data = e.data.json();
  e.waitUntil(self.registration.showNotification(data.title, {
    body: data.body,
    tag: data.tag,            // схлопывание дублей по question_id
    data: { url: data.url },  // диплинк на карточку вопроса
  }));
});
self.addEventListener('notificationclick', (e) => {
  e.notification.close();
  e.waitUntil(clients.openWindow(e.notification.data.url));
});
```

## 6. Экраны

### 6.1. Главный — «Сегодня»
- Сверху крупный счётчик-сводка: Открыто / Срочно / Жду ответа.
- Лента открытых вопросов, сортировка: urgent → просроченные напоминания → остальные по `remind_at`.
- Карточка вопроса: автор, суть, бейдж статуса, «висит N ч/дн», для waiting — «жду: {waiting_for}».
- Свайпы по карточке: вправо — «Закрыть» (открывает мини-форму resolution), влево — «Отложить» (пресеты: +1 ч, +4 ч, завтра утром).
- Плавающая кнопка «+» — всегда на экране.

### 6.2. Быстрый ввод (модалка поверх любого экрана)
- Одно большое текстовое поле с автофокусом. Плейсхолдер: «Иванова из бухгалтерии — не работает выгрузка в 1С».
- Под полем три таб-чипа приоритета: 🔥 Срочно / ◯ Обычно / ・Потом (по умолчанию «Обычно»).
- Кнопка «Сохранить» — и всё, модалка закрывается, вопрос в ленте, `remind_at` проставлен по правилам. Цель — **5 секунд от открытия до сохранения**.
- Поле «Кто спросил» — опционально, парсится эвристикой из текста (всё до первого тире/запятой предлагается как автор, редактируемо).

### 6.3. Карточка вопроса
- Полный текст, автор, приоритет, статус (переключение таб-баром), история из `activity_log` таймлайном.
- Для `waiting` — поле «От кого ждём».
- Кнопка «Закрыть вопрос» → обязательное поле «Как решили» (resolution) → статус closed.
- Кнопка «Напомнить в…» — ручной выбор даты/времени поверх автоматики.

### 6.4. Архив
- Закрытые вопросы, группировка по дням, поиск по тексту.

### 6.5. Отчёт
- Выбор периода: сегодня / эта неделя / произвольный диапазон.
- Превью статистики: пришло / закрыто / среднее время закрытия / топ-авторы вопросов.
- Кнопка «Скачать PDF» и «Поделиться» (`navigator.share` с файлом — на iOS откроется шторка, можно сразу в Telegram/почту руководителю).

### 6.6. Настройки
- Рабочее окно пушей (часы, дни).
- Вечерний чек вкл/выкл.
- Статус push-подписки + кнопка «Отправить тестовый пуш».
- Экспорт всех данных в JSON.

## 7. PDF-отчёт (`@react-pdf/renderer`)

Структура документа (А4, портрет):

1. **Шапка:** «Отчёт по работе с обращениями», ФИО, период, дата формирования. Тонкая пастельная плашка, без логотипов.
2. **Сводка** (4 показателя крупными цифрами): поступило · закрыто · в работе на конец периода · среднее время закрытия.
3. **Таблица закрытых вопросов:** дата, автор, суть, решение (resolution), время в работе. Зебра-строки очень светлым тоном.
4. **Открытые на конец периода:** отдельная короткая таблица с указанием статуса и «висит N дней» — это честно показывает руководителю текущую загрузку.
5. **Подвал:** «Сформировано автоматически · Кико» мелким серым.

Шрифт — кириллический обязательно (например, `Noto Sans` / `PT Sans`, зарегистрировать через `Font.register`, файлы .ttf положить в `public/fonts`). Без регистрации кириллица в react-pdf превратится в кракозябры — это известные грабли.

## 8. Дизайн-направление: «Koneko style»

Эстетика: японский минимализм, пастель, много воздуха. Приложение должно быть **приятным ежедневным инструментом**, а не корпоративной тикетницей.

### Палитра
- Фон: тёплый почти-белый `#FBFAF7` (бумага васи).
- Акцент: пыльно-розовый сакуры `#E8B4B8` — primary-кнопки, активные состояния.
- Вторичный: шалфейно-зелёный `#A8C3A0` — статус «закрыто», позитивные состояния.
- Срочность: приглушённый терракот `#D98E73` (не кричащий красный).
- Текст: тёплый графит `#3A3733`, вторичный `#8B857C`.
- Статусы бейджами в пастельных заливках 10–15% opacity с текстом полного тона.

### Типографика и формы
- Шрифт: `Plus Jakarta Sans` (заголовки semibold, текст regular). Никаких Inter/Roboto.
- Иконки: Phosphor (weight="light") — тонкие, точные линии.
- Карточки: техника «двойного безеля» — внешняя обёртка `bg-black/[0.03] ring-1 ring-black/5 p-1.5 rounded-[1.75rem]`, внутреннее ядро `bg-white rounded-[calc(1.75rem-0.375rem)]` с очень мягкой рассеянной тенью (`shadow-[0_8px_30px_rgba(58,55,51,0.06)]`). Никаких резких `shadow-md` и серых 1px-рамок.
- Кнопки — полные пилюли `rounded-full`, primary с иконкой в собственном круглом «острове» справа.
- Один деликатный японский графический акцент: тонкий контурный энсо/веточка сакуры как watermark на пустых состояниях («Все вопросы закрыты 🌸 Отдыхай») и в шапке отчёта. Без перегруза — один элемент на экран максимум.

### Моушен
- Все переходы `duration-500 ease-[cubic-bezier(0.32,0.72,0,1)]` — никаких `linear`/`ease-in-out`.
- Свайп карточки — с физикой возврата (spring), закрытие вопроса — карточка мягко схлопывается по высоте + короткий haptic-паттерн через `navigator.vibrate` не поддерживается на iOS, поэтому вместо него — микроанимация «лепесток улетает».
- Появление элементов ленты — staggered fade-up (`translate-y-4 opacity-0 → 0/100`, задержка 40 мс на элемент), через IntersectionObserver.
- Анимировать только `transform`/`opacity`. `backdrop-blur` — только на фиксированных элементах (модалки, таб-бар).

### Мобильная дисциплина
- Всё проектируется mobile-first под ~390px. `min-h-[100dvh]`, не `h-screen` (iOS Safari прыгает).
- Нижний таб-бар: Сегодня / Архив / Отчёт / Настройки — плавающая glass-пилюля с отступом от края, safe-area учтена (`pb-[env(safe-area-inset-bottom)]`).
- Тач-таргеты ≥ 44px.

## 9. Безопасность и приватность

- Supabase Auth, magic link, один пользователь. RLS-политики `auth.uid()` на все таблицы.
- Все ключи — в env (`.env.local` в `.gitignore`, на Vercel — Environment Variables). **Никаких токенов и ключей в коде** — ни VAPID private, ни service key.
- Edge Functions используют service key только на сервере.
- В вопросах может быть чувствительная рабочая информация — в README проекта пометка: не хранить в raw_text пароли/персональные данные, которые нельзя выносить за периметр компании.

## 10. Этапы (для Claude Code)

**Этап 1 — каркас (день 1):**
Vite + React + TS + Tailwind, vite-plugin-pwa (injectManifest), манифест (имя «Кико», иконки, `display: standalone`, тема `#FBFAF7`), Supabase-клиент, схема БД миграцией, Auth magic link, деплой на Vercel. Проверка: ставится на iPhone как PWA.

**Этап 2 — ядро (день 1–2):**
Быстрый ввод, лента «Сегодня», карточка вопроса, статусы, свайпы, архив, activity_log. Дизайн-система по разделу 8.

**Этап 3 — пуши (день 2):**
VAPID-ключи, подписка на клиенте, кастомный SW, таблица подписок, Edge Function `send-reminders` + `morning-digest`, pg_cron, тестовый пуш из настроек. Проверка на реальном iPhone (симулятор пуши в PWA не покажет).

**Этап 4 — отчёт (день 3):**
Экран отчёта, статистика, PDF с кириллическим шрифтом, `navigator.share`.

**Этап 5 — полировка:**
Онбординг установки PWA, пустые состояния с сакурой, экспорт JSON, моушен-проходка по всем экранам.

**v2 (после обкатки, не делать сейчас):**
AI-парсинг ввода через Anthropic API (автор/суть/приоритет из свободного текста), голосовой ввод через Web Speech API, недельная AI-выжимка для отчёта («главные темы недели»).

## 11. Критерии приёмки

- [ ] От тапа «+» до сохранённого вопроса — ≤ 5 секунд, 2 действия (ввод + сохранить).
- [ ] Пуш приходит на заблокированный iPhone при установленной PWA.
- [ ] Открытый вопрос невозможно «потерять»: он напоминает о себе, пока не закрыт.
- [ ] Закрытие вопроса невозможно без заполнения resolution.
- [ ] PDF-отчёт за неделю с корректной кириллицей генерится и шарится в один тап.
- [ ] Пуши не приходят ночью и в выходные (в дефолтных настройках).
- [ ] Ни одного секрета в репозитории.
